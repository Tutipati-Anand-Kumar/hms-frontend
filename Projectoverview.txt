PROJECT OVERVIEW
================================================================================
MSCureChain is a comprehensive Hospital Management System designed to streamline
healthcare operations for hospitals, doctors, patients, and administrative staff.
The system provides digital solutions for OPD management, appointment booking,
prescription management, and patient record keeping.

TECHNOLOGY STACK
================================================================================
Frontend:
- React.js (with React Router for navigation)
- Tailwind CSS (for styling)
- Lucide React (for icons)
- React Hot Toast (for notifications)
- Axios (for API calls)

Backend:
- Node.js with Express.js
- MongoDB (database)
- JWT (authentication)

Theme System:
- CSS Variables for dynamic dark/light theme switching
- Theme stored in localStorage
- Automatic theme persistence across sessions


USER ROLES & ACCESS
================================================================================

1. PATIENT
   - Access: Patient Dashboard (/home)
   - Capabilities: Book appointments, view medical records, manage prescriptions

2. DOCTOR
   - Access: Doctor Dashboard (/doctor)
   - Capabilities: Manage appointments, create prescriptions, view patients

3. ADMIN / SUPER-ADMIN
   - Access: Admin Dashboard (/admin)
   - Capabilities: Manage users, hospitals, doctors, helpdesk staff

4. HELPDESK
   - Access: HelpDesk Dashboard (/helpdesk)
   - Capabilities: Manage appointments, assist patients, create doctors

AUTHENTICATION & AUTHORIZATION
================================================================================

Authentication Flow:
1. User enters credentials on login page
2. Backend validates credentials
3. JWT tokens (access + refresh) are generated
4. Tokens stored in sessionStorage
5. User redirected based on role

Authorization:
- Private routes protected by PrivateRoutes component
- Role-based access control (RBAC)
- Allowed roles specified per route
- Unauthorized access redirects to login

Token Management:
- Access token: Short-lived, used for API requests
- Refresh token: Long-lived, used to get new access token
- Stored in sessionStorage
- Cleared on logout
